<objective>
Replace the spherical scattered position distribution with a directional tunnel-shaped ellipsoid so the dissolve reads as "falling into a dream corridor" instead of a generic spherical explosion (D-02). </objective>

<action>
Modify `src/components/particleMemory/particleSampler.js` to replace the spherical golden-ratio distribution with an elongated ellipsoid aligned to the Z-axis (camera travel axis).

Replace the scattered position generation (lines 174-181) with a directional tunnel distribution:

1. The tunnel axis is Z (camera travel direction). Stretch the Z-axis by a `tunnelStretch` factor (default 3.5) so particles are distributed in a cigar-shaped volume.
2. Use the same golden-ratio spiral for angular distribution (deterministic, avoids clustering).
3. Radial distance uses a power-law distribution biased toward the tunnel walls (not uniformly filled) to create visible layered depth bands.
4. Add a forward bias: particles slightly offset toward positive Z (the "destination" direction).
5. Read `scatteredRadius` and new `tunnelStretch` from config (with fallbacks).
</action>

<task id="16-01-02" title="Add uMorphStagger uniform to particle shaders">
<read_first>
- src/components/particleMemory/particleShaders.js (vertex shader, uMorphProgress uniform)
- .planning/phases/16-dream-portal-transition/16-CONTEXT.md (D-10: depth-staggered convergence)
</read_first>
<action>
Modify `src/components/particleMemory/particleShaders.js` to add a `uMorphStagger` uniform that offsets the morph progress per-particle based on depth. This creates depth-staggered convergence where foreground particles (low aDepthValue) converge first during reform.

1. Add `uniform float uMorphStagger;` to the vertex shader uniforms (after uMorphProgress).
2. Compute an effective morph progress per-particle:
   ```glsl
   float staggerOffset = aDepthValue * uMorphStagger;
   float effectiveMorph = clamp(uMorphProgress - staggerOffset, 0.0, 1.0);
   ```
3. Replace the existing `mix(aScatteredPosition, aPhotoPosition, uMorphProgress)` with `mix(aScatteredPosition, aPhotoPosition, effectiveMorph)`.

When uMorphStagger is 0.0 (default), behavior is unchanged. When uMorphStagger is positive (e.g. 0.3), deeper particles lag behind during morph transitions.
</action>
<acceptance_criteria>
- grep `uMorphStagger` in particleShaders.js returns at least 2 matches (uniform decl + usage)
- effectiveMorph is clamped to [0, 1]
- When uMorphStagger = 0.0, behavior is identical to previous code
- Depth-based stagger: higher aDepthValue = delayed morph (background lags foreground)
</acceptance_criteria>
</task>
