Create `src/utils/weather.js`:
- `fetchWeather()`: Use browser `navigator.geolocation.getCurrentPosition` to get lat/lon, then fetch from `https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}&current=temperature_2m,weather_code,cloud_cover,wind_speed_10m,precipitation,is_day`. Return parsed JSON.
- Cache strategy: Store result in localStorage key `jarowe_weather_cache` as `{ data, timestamp, lat, lon }`. On subsequent calls, return cache if < 30 minutes old. If geolocation denied or fetch fails, return null (caller handles fallback).
- `applyWeatherAtmosphere(weatherData)`: If weatherData is null, apply defaults (clear day) and return. Otherwise set CSS custom properties on `:root`:
  - `--weather-fog-density`: 0.0 (clear) to 1.0 (fog/mist) based on weather_code (45/48 = fog codes)
  - `--weather-particle-speed`: 0.5 (calm) to 2.0 (storm) based on wind_speed_10m
  - `--weather-precipitation`: 0.0 to 1.0 based on precipitation mm
  - `--weather-color-warmth`: -0.5 (cold/blue shift below 5C) to 0.5 (warm/amber shift above 30C) based on temperature_2m
  - `--weather-cloud-opacity`: 0.0 to 0.8 based on cloud_cover percentage
- `getWeatherUniforms(weatherData)`: Returns object with numeric values suitable for globe shader uniforms: `{ uFogDensity, uWindSpeed, uPrecipitation, uCloudOpacity }`. Returns zeros if weatherData is null.
- These CSS properties are ADDITIVE to --tod-* properties. They do NOT replace time-of-day. The CSS should multiply/layer (e.g., fog reduces visibility on top of existing atmosphere).
- Weather codes reference: https://open-meteo.com/en/docs (WMO code table). Key codes: 0=clear, 1-3=partly cloudy, 45/48=fog, 51-67=drizzle/rain, 71-77=snow, 80-82=rain showers, 95-99=thunderstorm.

Create `src/utils/easterEggs.js`:
- Define `SITE_BIRTHDAY` constant (export it) — set to `{ month: 2, day: 28 }` (Feb 28, v1.0 ship date from ROADMAP).
- `checkEasterEggs(date = new Date())`: Returns `{ event: string, cssClass: string, glintDialogue: { text: string, expression: string }[] } | null`
- Check these 5 events in priority order (return first match):
  1. **Full Moon**: Use `getMoonPhase(date) === 'full'` from astro.js. CSS class: `easter-egg--full-moon`. Effects: enhanced glow/particles.
  2. **Friday the 13th**: `date.getDay() === 5 && date.getDate() === 13`. CSS class: `easter-egg--friday13`. Effects: spooky shift.
  3. **Pi Day**: Month=3, Day=14. CSS class: `easter-egg--pi-day`. Effects: math-themed.
  4. **Summer Solstice** (June 20-21) / **Winter Solstice** (Dec 21-22): CSS class: `easter-egg--solstice-summer` or `easter-egg--solstice-winter`. Effects: golden boost or blue boost.
  5. **Site Birthday**: Match SITE_BIRTHDAY month/day. CSS class: `easter-egg--birthday`. Effects: celebration confetti.
- Each event returns 3-4 Glint dialogue lines (text + expression) themed to the event.
- CSS classes are designed to be added to `document.body.classList` additively alongside `data-tod-phase`.
