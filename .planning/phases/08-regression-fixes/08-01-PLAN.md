In `src/constellation/scene/NodeCloud.jsx`, the `meshStandardMaterial` has `emissive="#444466"` -- a static grey-blue applied uniformly to all instances. Three.js `instanceColor` only modulates `diffuseColor` in the fragment shader, NOT `totalEmissiveRadiance`. This means every node emits the same glow regardless of theme.

**Fix:** Use `onBeforeCompile` on the MeshStandardMaterial to inject shader code that multiplies `totalEmissiveRadiance` by `vColor.rgb` (the per-instance color varying). This makes each node's emissive glow match its theme color.

1. Change the material's `emissive` prop from `"#444466"` to `"#ffffff"` -- this provides a neutral white base that will be tinted by the per-instance color injection.

2. Add an `onBeforeCompile` callback to the `meshStandardMaterial` JSX element. The callback receives `(shader)` and must:
   - In `shader.fragmentShader`, find the line `#include <emissivemap_fragment>`
   - After that include, append this GLSL: `totalEmissiveRadiance *= vColor.rgb;`
   - This multiplies the white emissive by the per-instance color, producing theme-colored glow

3. The `onBeforeCompile` callback must also set `materialRef.current.customProgramCacheKey = () => 'node-cloud-emissive-instanced';` to prevent Three.js from sharing the compiled shader program with unpatched MeshStandardMaterials. Actually, use the `customProgramCacheKey` prop on the JSX element: add a function that returns a unique string.

4. Wrap the `onBeforeCompile` function in `useCallback` with an empty dependency array so the material does not recompile every render.

Concrete changes to the JSX (replacing lines 267-275):

```jsx
<meshStandardMaterial
  ref={materialRef}
  color="#ffffff"
  emissive="#ffffff"
  emissiveIntensity={getCfg('nodeEmissiveIntensity')}
  roughness={0.6}
  metalness={0.1}
  toneMapped={false}
  onBeforeCompile={handleShaderCompile}
/>
```

Add above the component (or inside it with useCallback):

```javascript
const handleShaderCompile = useCallback((shader) => {
  shader.fragmentShader = shader.fragmentShader.replace(
    '#include <emissivemap_fragment>',
    `#include <emissivemap_fragment>
     totalEmissiveRadiance *= vColor.rgb;`
  );
}, []);
```

Add `useCallback` to the import from 'react' at the top of the file (line 1 currently imports `useRef, useMemo, useEffect`).

5. No other files need changing. The `getNodeColor()` function and `setColorAt()` calls already set correct per-instance colors -- those colors just weren't reaching the emissive channel.
