    **2. Update `src/context/AudioContext.jsx` -- Add music control listener:**

    Inside the AudioProvider component, add a useEffect that listens for `glint-action` events with `control_music` action:
    ```javascript
    // Listen for glint-action events for music control
    useEffect(() => {
      const handleGlintAction = (e) => {
        const { action, params } = e.detail || {};
        if (action === 'control_music' && params?.action) {
          if (params.action === 'play' && !isPlaying) togglePlay();
          else if (params.action === 'pause' && isPlaying) togglePlay();
          else if (params.action === 'next') handleNext();
        }
      };
      window.addEventListener('glint-action', handleGlintAction);
      return () => window.removeEventListener('glint-action', handleGlintAction);
    }, [isPlaying, togglePlay, handleNext]);
    ```

    Note: `isPlaying`, `togglePlay`, and `handleNext` are already defined in AudioProvider scope. The dependency array must include them. `togglePlay` and `handleNext` should be wrapped in useCallback if they aren't already -- check the file. If they are plain functions (not useCallback), they will cause the effect to re-register on every render. This is acceptable for an event listener -- the old one is cleaned up and new one attached. But if performance is a concern, wrap them in useCallback. Based on the grep output, they are plain functions -- leave as-is for now.

    } else if (action === 'show_daily') {
