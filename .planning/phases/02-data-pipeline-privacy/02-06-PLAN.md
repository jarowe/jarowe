**store.js -- Add data fields and loadData action:**

Add these fields to the existing Zustand store (do NOT remove any existing fields -- viewMode, gpuTier, focusedNodeId, hoveredNodeIdx, filterEntity, lightbox, timelinePosition all stay):

```js
// Data fields (initialized empty -- loaded on mount)
nodes: [],
edges: [],
epochs: [],
dataLoading: true,
dataError: null,

// Load data action
loadData: async () => {
  // Prevent double-load
  const state = get();
  if (state.nodes.length > 0 && !state.dataError) return;

  set({ dataLoading: true, dataError: null });
  try {
    const { loadConstellationData } = await import('../constellation/data/loader.js');
    const data = await loadConstellationData();
    set({
      nodes: data.nodes,
      edges: data.edges,
      epochs: data.epochs,
      dataLoading: false,
    });
  } catch (err) {
    console.error('Failed to load constellation data:', err);
    set({ dataError: err.message, dataLoading: false });
  }
},
```

Change the store creator from `create((set) => ({` to `create((set, get) => ({` to support the `get()` call in loadData.

**IMPORTANT:** The dynamic `import()` of loader.js is intentional. It keeps the loader (and its mock-constellation.json fallback import) out of the initial bundle until data is actually needed. This matches the existing code-splitting pattern used elsewhere in the project.
