# Plan 14-01: Particle Field Renderer + CapsuleShell Integration

**Phase:** 14-particle-field-core
**Requirements:** INTEG-01, INTEG-02, PART-01, PART-03 (partial), PART-04 (partial)

## Goal

Build the core particle field renderer that samples a photo + depth map into 80K-150K luminous 3D particles with dual position buffers (scattered + photo-formed), breathing animation, and bloom glow -- integrated into CapsuleShell dispatch as `renderMode: 'particle-memory'`.

## Tasks

### Task 1: ParticleMemoryField component with CPU-side particle sampling

**read_first:**
- `src/pages/CapsuleShell.jsx` (AtmosphericParticles pattern, DisplacedMeshRenderer pattern)
- `src/data/memoryScenes.js` (scene registry)
- `src/utils/gpuCapability.js` (tier system)

**action:**
Create `src/pages/ParticleMemoryField.jsx` containing:
1. CPU-side particle sampling function: loads photo + depth as Image, draws to offscreen canvas, getImageData(), computes Float32Arrays for positions, colors, sizes
2. Hybrid grid + edge boost sampling: base ~80K UV grid + Sobel edge detection on depth for ~40-70K bonus particles at depth discontinuities
3. Dual position buffers: `aPhotoPosition` (photo-formed) and `aScatteredPosition` (random scattered) as BufferGeometry attributes
4. Particle vertex shader with soft-circle fragment shader, depth-correlated breathing, size variation (depth + luminance driven)
5. Shader-based halo glow (extended soft-circle falloff with secondary low-alpha ring)
6. `morphProgress` uniform for future Phase 16 dissolve/reform (default 1.0 = photo-formed)

**acceptance_criteria:**
- ParticleMemoryField.jsx exists and exports a React component
- Component loads photo + depth via offscreen canvas and produces Float32Array attributes
- Both `aScatteredPosition` and `aPhotoPosition` attributes present in BufferGeometry
- Vertex shader interpolates between scattered and photo positions using `uMorphProgress`
- Fragment shader renders soft circles with halo glow
- Breathing animation driven by depth value (foreground breathes first)

### Task 2: Wire connections (LineSegments) + tier adaptation

**read_first:**
- `src/pages/ParticleMemoryField.jsx` (from Task 1)
- 14-CONTEXT.md D-05, D-06, D-07 (wire connection decisions)

**action:**
Add to ParticleMemoryField.jsx:
1. Spatial hash grid for efficient neighbor lookup
2. Wire connection computation: dense connections at depth edges (high Sobel gradient + distance threshold) + sparse KNN ambient connections
3. THREE.LineSegments with ShaderMaterial -- emissive color averaged from connected particles, alpha fading with distance
4. Static topology, dynamic alpha (endpoints follow breathing, alpha can pulse)
5. Tier adaptation:
   - Full tier: 150K particles, wire connections, shader halo + postprocessing bloom-ready, DPR up to 2.0
   - Simplified tier: 50-60K particles (grid only), NO wires, shader halo only, DPR 1.0
   - Parallax tier: existing ParallaxFallback + sparse CSS-animated particle overlay

**acceptance_criteria:**
- Wire connections render as thin luminous LineSegments along depth contours
- Spatial hash used for neighbor lookup (not O(n^2))
- Simplified tier skips wire connections and edge-boost particles
- Connection count ~10K for full tier
