# Plan 15-01: Flight Camera + Scroll-Driven Spline Controller

**Phase:** 15-memory-flight-controller
**Requirements:** FLIGHT-01, FLIGHT-02, FLIGHT-03, FLIGHT-04

## Goal

Build a scroll-driven flight camera that moves along a CatmullRom spline through the particle field with spring-smoothed momentum, inertial drift, micro-drift breathing, mouse parallax, FOV narrowing, and progress-threshold narrative card triggers -- replacing the GSAP-driven CinematicCamera for particle-memory scenes.

## Tasks

### Task 1: FlightCamera component with CatmullRom spline + scroll input

**read_first:**
- `src/pages/CapsuleShell.jsx` (CinematicCamera export, lines 210-310)
- `src/data/memoryScenes.js` (cameraKeyframes format)
- 15-CONTEXT.md (D-01 through D-08)

**action:**
Create `src/components/particleMemory/FlightCamera.jsx` containing:
1. CatmullRom spline built from scene flightPath keypoints (5-7 points)
2. Spring-smoothed scroll progress: raw wheel delta → velocity accumulator → spring-damped 0→1 progress float
3. Exponential decay on scroll release (0.95 per frame) for "gliding" sensation
4. Touch support via pointer events (Y-delta maps to same velocity accumulator)
5. FOV narrowing: 50deg at start → ~40deg at deepest point → 50deg at end, driven by progress
6. Mouse parallax (15-30ms response) layered on spline position
7. Sine-wave micro-drift (8-20s period, tiny amplitude) when user stops scrolling
8. Hold at final keypoint when progress reaches 1.0, with gentle breathing drift
9. Expose `progress` value via useImperativeHandle for downstream consumers (narrative, soundscape)

**acceptance_criteria:**
- FlightCamera.jsx exists and exports a forwardRef R3F component
- Camera position interpolated along CatmullRom spline based on spring-smoothed progress
- Scroll wheel moves camera forward/backward along spline
- Touch swipe (Y-delta) drives same velocity accumulator
- Exponential velocity decay creates visible gliding after scroll release
- FOV narrows at midpoint and widens at ends
- Mouse parallax offsets camera position subtly
- Micro-drift sine wave active when velocity is near zero
- `progress` (0-1) exposed via ref

### Task 2: memoryScenes.js flightPath config for syros-cave

**read_first:**
- `src/data/memoryScenes.js` (syros-cave entry)
- Task 1 FlightCamera.jsx (expected flightPath format)

**action:**
Add `flightPath` configuration to the syros-cave scene entry in memoryScenes.js:
1. `flightPath.keypoints`: Array of 6 {x,y,z} points defining the CatmullRom spline through the particle field
   - Start: outside the field, looking at it from a distance
   - Middle: sweep past the surface, dive into the interior
   - End: emerge on the other side, settle
2. `flightPath.lookAtKeypoints`: Corresponding look-at targets for each keypoint
3. `flightPath.fovRange`: [50, 40] (start/min FOV)
4. `flightPath.fovCurve`: bell-shaped progress mapping (FOV narrows at ~0.5 progress)
5. `narrativeThresholds`: Array of { progress: number, text: string } replacing time-based delays
   - Map existing 4 narrative cards to progress thresholds: 0.15, 0.35, 0.6, 0.85

**acceptance_criteria:**
- syros-cave scene has `flightPath` object with keypoints, lookAtKeypoints, fovRange
- `narrativeThresholds` array present with 4 entries at different progress values
- Existing cameraKeyframes preserved (used by CinematicCamera on non-flight scenes)
- No other scenes affected
