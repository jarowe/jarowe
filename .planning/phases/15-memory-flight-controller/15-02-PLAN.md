# Plan 15-02: Progress-Threshold Narrative Triggers

**Phase**: 15 — Memory Flight Controller
**Requirements**: FLIGHT-03 (narrative driven by progress, not time)
**Estimated tasks**: 2

## Context

Phase 15-01 (parallel) adds `flightPath` to syros-cave in memoryScenes.js and creates
`flightProgressRef` in CapsuleShell. This plan adds `threshold` fields to narrative
entries and reads `flightProgressRef` in CapsuleShell to trigger cards at progress
thresholds instead of time-based delays for particle-memory scenes.

## Tasks

### Task 1: Add threshold fields to narrative entries in memoryScenes.js

**Action**: Add `threshold` field (0-1 float) to each narrative entry in the
syros-cave scene. Keep `delay` for backward compatibility with non-particle-memory
scenes (placeholder-scene, test-capsule). For syros-cave, both `delay` (fallback)
and `threshold` (progress-based) coexist.

**Read first**: `src/data/memoryScenes.js`

**Thresholds**:
- Card 1 (Place): threshold 0.15
- Card 2 (Feeling): threshold 0.40
- Card 3 (Meaning): threshold 0.65
- Card 4 (Gratitude): threshold 0.85

**Acceptance criteria**:
- syros-cave narrative entries each have a `threshold` field (number 0-1)
- placeholder-scene and test-capsule narrative entries are unchanged
- File exports correctly (no syntax errors)

### Task 2: Add progress-threshold trigger path in CapsuleShell.jsx

**Action**: In the narrative card useEffect, add a branch for particle-memory scenes
whose narrative entries have `threshold` fields. When `scene.renderMode === 'particle-memory'`
and entries have thresholds, poll a `flightProgressRef` (exposed by ParticleFieldRenderer
or the flight controller) and trigger cards when progress crosses each threshold.
Preserve time-based narrative for all other scene types.

**Read first**: `src/pages/CapsuleShell.jsx`

**Implementation**:
- Add a `flightProgressRef` via useRef(0) in CapsuleShell
- Pass it as a prop to ParticleFieldRenderer (the flight controller will write to it)
- In the narrative useEffect, detect if narrative entries have `threshold` fields
- If thresholds: use a requestAnimationFrame loop to poll `flightProgressRef.current`
  and trigger cards when progress >= threshold
- If no thresholds (or non-particle-memory): preserve existing time-based delay logic
- Track which thresholds have already fired to prevent re-triggering

**Acceptance criteria**:
- CapsuleShell has `flightProgressRef` ref
- `flightProgressRef` is passed as prop to ParticleFieldRenderer
- Narrative useEffect has two code paths: threshold-based (particle-memory) and time-based (other)
- Time-based narrative for non-particle-memory scenes is unchanged
- No build errors
