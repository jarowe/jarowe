# Plan 18-01: Rubric Schema + Scoring Dimensions

## Goal

Add the 5-dimension grading rubric to the lab UI and CLI so every world-generation family is scored on the same scale. Implement weighted composite calculation, provenance on every evaluation, and wire the rubric into both the lab API and grade-memory-world.mjs.

```js
function computeWeightedComposite({ coherence, exploration, subject, artifacts, emotion }) {
  if ([coherence, exploration, subject, artifacts, emotion].some(v => v == null)) return null;
  return +(coherence * 0.25 + exploration * 0.15 + subject * 0.25 + artifacts * 0.15 + emotion * 0.20).toFixed(2);
}
```

4. In the `review` object built in `main()`, extend `humanReview` to include:
```js

    explorationRange: exploration,
    subjectPreservation: subject,
    artifactSeverity: artifacts,
    emotionalRead: emotion,
    composite: computeRawComposite(scores),
    weightedComposite: computeWeightedComposite(scores),
  } : null,
  evaluator,
  notes: notes ?? null,
  favorite,
}
```

5. In the index entry pushed to `index.json`, add `rubricWeightedComposite` and `evaluator` fields alongside existing `grade` and `machineScore`.

6. When `--compare` flag is set, implement comparison mode:
   - Read `versions/index.json` for the scene
   - Group versions by `worldFamily`
   - For each family, find the version with the highest `rubricWeightedComposite` (or `grade` if no rubric)
   - Output a JSON comparison summary to stdout AND write to `world/versions/comparison.json`:
   ```json
   {
     "sceneId": "naxos-rock",
     "comparedAt": "2026-04-05T...",
     "families": [
       {
         "family": "marble",
         "bestVersion": "...",

         "dealBreaker": null

     "winner": "marble",

     "tieBreaker": null

   ```
   - Winner determination: highest weightedComposite wins. If within 0.2, use emotionalRead tiebreaker per GRADING-RUBRIC.md Section B.6.

7. Update the usage help string (printed when no sceneId) to include rubric flags and dimension descriptions.
</action>
